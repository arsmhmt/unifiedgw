What is a Client Site?

A client site is a registered website/application that uses the bank gateway system to process payments.

API Key
  • Unique identifier for API authentication
  • Generated automatically when site is created
  • Can be regenerated if compromised
  • Must be kept secure and never shared publicly
Callback & Webhook URLs
  • Callback URL: Receives payment status updates
  • Webhook URL: Receives real-time event notifications
  • Both URLs must be publicly accessible
  • Should use HTTPS for security
Transaction Limits
  • Set minimum and maximum amounts for deposits/withdrawals
  • Prevents fraudulent or irregular transactions
  • Can be adjusted based on client needs
Important: After creating the site, provide the API key to the client securely. It will only be displayed once.
