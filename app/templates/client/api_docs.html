Getting Started

PayCrypt provides two main API systems for accepting payments:

Crypto Payment API

Accept cryptocurrency payments with hosted checkout pages and real-time notifications.

  • HMAC-signed requests for security
  • Hosted checkout pages
  • Real-time webhooks
  • Multiple cryptocurrencies
Bank Gateway API

Process traditional bank payments and transfers with multiple banking partners.

  • Simple API key authentication
  • Deposit and withdrawal requests
  • Real-time status updates
  • Multiple bank accounts
Base URLs
  • Crypto API: {{ request.host_url }}api/v1/
  • Bank API: {{ request.host_url }}bank-api/

Crypto Payment API

Authentication

All crypto API requests must include HMAC authentication headers:

Required Headers
X-Paycrypt-Key Your API key from the dashboard
X-Paycrypt-Timestamp Unix timestamp in seconds (UTC)
X-Paycrypt-Signature HMAC-SHA256 signature of "."
Content-Type application/json
Signature Generation
// JavaScript const timestamp = Math.floor(Date.now() / 1000).toString(); const message = `${timestamp}.${JSON.stringify(requestBody)}`; const signature = crypto.createHmac('sha256', secretKey) .update(message) .digest('hex'); # Python import hmac import hashlib import time import json timestamp = str(int(time.time())) message = f"{timestamp}.{json.dumps(request_body)}" signature = hmac.new( secret_key.encode(), message.encode(), hashlib.sha256 ).hexdigest()

Endpoints

POST /api/v1/payment_sessions Create Payment Session

Create a new payment session for cryptocurrency payments.

Request Body:
{ "order_id": "ORD-12345", "amount": "99.99", "currency": "USD", "customer": { "email": "customer@example.com" }, "metadata": { "source": "your_app", "user_id": "12345" }, "success_url": "https://yoursite.com/payment/success?order=ORD-12345", "cancel_url": "https://yoursite.com/payment/cancel?order=ORD-12345", "webhook_url": "https://yoursite.com/api/paycrypt/webhook" }
Response (201 Created):
{ "id": "ps_abcd1234ef56", "status": "created", "checkout_url": "https://checkout.paycrypt.online/ps_abcd1234ef56", "expires_at": 1694692800, "order_id": "ORD-12345", "amount": "99.99", "currency": "USD" }
GET /api/v1/payment_sessions/{id} Get Payment Session

Retrieve the current status of a payment session.

Response (200 OK):
{ "id": "ps_abcd1234ef56", "status": "completed", "order_id": "ORD-12345", "amount": "99.99", "currency": "USD", "paid_at": 1694692900, "transaction_hash": "0x1234567890abcdef..." }
POST /api/v1/betslip/generate Generate Betslip (Gaming)

Generate a betslip for gaming and sports betting platforms.

Request Body:
{ "betslip_id": "BS-12345", "user_id": "user123", "bets": [ { "event_id": "match_001", "bet_type": "winner", "odds": 2.5, "stake": "50.00" } ], "total_stake": "50.00", "potential_payout": "125.00", "webhook_url": "https://yoursite.com/api/betslip/webhook" }

Webhooks

PayCrypt sends webhook notifications to your specified URL when payment events occur.

Webhook Events
payment.created Payment session was created
payment.completed Payment was successfully completed
payment.failed Payment failed or was rejected
payment.expired Payment session expired
Webhook Verification
# Python webhook verification import hmac import hashlib def verify_webhook(request): signature = request.headers.get('X-Paycrypt-Signature') timestamp = request.headers.get('X-Paycrypt-Timestamp') body = request.body.decode('utf-8') expected_signature = hmac.new( webhook_secret.encode(), f"{timestamp}.{body}".encode(), hashlib.sha256 ).hexdigest() return hmac.compare_digest(signature, expected_signature)

Bank Gateway API

Authentication

Bank gateway API uses simple API key authentication:

Required Headers
X-API-Key Your bank gateway API key
Content-Type application/json

Endpoints

POST /bank-api/deposit/request Create Deposit Request

Create a new deposit request for bank transfer.

Request Body:
{ "amount": "500.00", "currency": "TRY", "user_name": "John Doe", "user_email": "john@example.com", "user_phone": "+90555123456", "callback_url": "https://yoursite.com/api/bank/callback", "external_transaction_id": "TXN-67890" }
Response (200 OK):
{ "success": true, "transaction_id": "TXN-67890", "reference_code": "REF-ABC123", "bank_details": { "bank_name": "Example Bank", "account_holder": "PayCrypt Gateway", "iban": "TR1234567890123456789012345", "amount": "500.00", "currency": "TRY" }, "instructions": "Please transfer exactly 500.00 TRY to the provided account.", "expires_at": "2025-09-15T10:00:00Z" }
POST /bank-api/withdraw/request Create Withdrawal Request

Create a new withdrawal request to user's bank account.

Request Body:
{ "amount": "200.00", "currency": "TRY", "user_name": "John Doe", "user_email": "john@example.com", "user_iban": "TR9876543210987654321098765", "callback_url": "https://yoursite.com/api/bank/callback", "external_transaction_id": "WTH-12345" }
GET /bank-api/transaction/{reference} Get Transaction Status

Check the status of a bank transaction.

Response (200 OK):
{ "success": true, "reference_code": "REF-ABC123", "status": "confirmed", "amount": "500.00", "currency": "TRY", "created_at": "2025-09-14T09:00:00Z", "confirmed_at": "2025-09-14T09:30:00Z", "transaction_type": "deposit" }

Callbacks

Bank gateway sends HTTP POST callbacks to your specified callback URL when transaction status changes.

Callback Payload
{ "reference_code": "REF-ABC123", "external_transaction_id": "TXN-67890", "status": "confirmed", "amount": "500.00", "currency": "TRY", "transaction_type": "deposit", "timestamp": "2025-09-14T09:30:00Z" }
