Webhook Events

Your webhook endpoint will receive the following events:

  • payment.created - New payment created
  • payment.pending - Payment is pending
  • payment.approved - Payment approved
  • payment.completed - Payment completed successfully
  • payment.failed - Payment failed
  • payment.rejected - Payment rejected
  • payment.cancelled - Payment cancelled
Webhook Headers

Each webhook request includes these headers:

  • X-Paycrypt-Event - Event type (e.g., payment.completed)
  • X-Paycrypt-Timestamp - ISO timestamp of the event
  • X-Paycrypt-Signature - HMAC-SHA256 signature for verification
  • X-Paycrypt-Event-Id - Unique event ID
Verifying Signatures

To verify webhook authenticity, compute HMAC-SHA256 of:

timestamp + "." + json_payload

Using your webhook secret as the key, then compare with the signature header.
