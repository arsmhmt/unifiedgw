Node.js USDT Deposit API Guide

This guide shows how to call the PayCrypt USDT deposit API from Node.js and receive webhooks when deposits are confirmed for your betting or casino application.

  • Create USDT deposits using fetch or axios.
  • Handle webhooks in an Express route.
  • Update balances only when status is confirmed.

1. Create a USDT deposit from Node.js

The example below uses the built-in fetch API available in recent Node.js versions. Replace <API_KEY> with your PayCrypt API key. 

const API_BASE = 'https://paycrypt.online';
const API_KEY = process.env.PAYCRYPT_API_KEY || '<API_KEY>';

async function createDeposit() {
  const res = await fetch(`${API_BASE}/api/v1/crypto/deposits`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      fiat_amount: 250,
      fiat_currency: 'USD',
      crypto_currency: 'USDT',
      crypto_network: 'TRC20',
    }),
  });

  console.log(res.status);
  const data = await res.json();
  console.log(data);
}

createDeposit().catch(console.error);

Store the response (especially payment_id, crypto_amount, and deposit_address) alongside your internal ticket or session.

2. Show deposit details in your app

In your cashier UI (React, Vue, server-rendered templates, etc.), render the returned address and QR code.

  • Persist the response JSON with your own player or bet ID.
  • Show the address and QR so players can send USDT.
  • Only credit the balance after receiving the webhook.

The exact stack is up to you; PayCrypt focuses on the deposit and confirmation layer.

3. Handle the webhook in Express

Configure your webhook URL in the PayCrypt dashboard, for example https://your-domain.com/webhooks/paycrypt/deposit. When a deposit is confirmed, PayCrypt sends a POST request with JSON to that URL. 

import express from 'express';

const app = express();
app.use(express.json());

app.post('/webhooks/paycrypt/deposit', (req, res) => {
  const data = req.body || {};

  const paymentId = data.payment_id;
  const status = data.status;
  const amount = data.amount;
  const currency = data.currency;

  // Respond quickly so PayCrypt treats the webhook as delivered.

In production you may want to add authentication, logging, and idempotency protection around this route.
